Buttons

Reusable button styles and interaction states for the LifeNote interface

Components

Overview

Variants

States

Purpose

Buttons support major user actions across the app. These styles create consistency for actions like navigation, journal creation, and feature access. States included: default, hover, pressed, and disabled.

Usage

Use primary buttons for the main action in a section.

Use secondary buttons for supporting actions.

Use accent buttons sparingly for emphasis.

Keep labels short and action-focused.
